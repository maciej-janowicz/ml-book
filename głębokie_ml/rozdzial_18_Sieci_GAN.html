"""

## 1. WPROWADZENIE DO SIECI GAN

### Co to są sieci GAN?

Generative Adversarial Networks (GAN) to architektura sztucznej inteligencji,
która składa się z dwóch sieci neuronowych "walczących" ze sobą:

1. **Generator** - tworzy sztuczne dane próbując oszukać dyskryminator
2. **Dyskryminator** - stara się odróżnić prawdziwe dane od sztucznych

Ta "gra" między sieciami prowadzi do powstania generatora, który potrafi tworzyć
bardzo realistyczne sztuczne dane.

### Dlaczego GAN są użyteczne dla ubezpieczycieli?

- **Augmentacja danych**: Tworzenie dodatkowych danych treningowych
- **Symulacja scenariuszy**: Modelowanie rzadkich zdarzeń ubezpieczeniowych
- **Ochrona prywatności**: Generowanie syntetycznych danych klientów
- **Wykrywanie oszustw**: Tworzenie przykładów fraudów do treningu modeli
- **Optymalizacja produktów**: Symulacja zachowań różnych segmentów klientów

Nieco dokładniej:

1. Augmentacja danych (szczególnie rzadkich przypadków oszustw)
   - W danych ubezpieczeniowych fraud to zjawisko rzadkie (np. 1–3% przypadków).
   - To powoduje, że klasyfikatory uczą się na bardzo niezrównoważonych zbiorach.
   - GAN-y mogą generować sztuczne próbki transakcji/fraudów, które wyglądają realistycznie
     i pozwalają trenować modele wykrywania oszustw.
   - Przykład: wygenerowanie dodatkowych 1000 „fałszywych wniosków o rentę”
     o cechach podobnych do znanych fraudów.

2. Symulacja nietypowych scenariuszy
   - GAN potrafi tworzyć „syntetyczne” dane odpowiadające rzadkim, ekstremalnym zdarzeniom.
   - W ubezpieczeniach może to być np.:
     - nietypowy schemat wyłudzeń w świadczeniach długoterminowych,
     - fałszywe faktury medyczne z drobnymi wariacjami,
     - nietypowe profile klientów w analizie ryzyka.
   To pozwala trenować systemy na oszustwach, które jeszcze się nie wydarzyły, ale są prawdopodobne.

3. Detekcja anomalii poprzez „dyskryminator”
   - W klasycznym GAN mamy generator i dyskryminator.
   - Dyskryminator uczy się odróżniać „realne dane” od „podrobionych”.
   - W zastosowaniu do oszustw można wykorzystać sam dyskryminator jako detektor anomalii:
   - Jeśli nowy wniosek wygląda „nienaturalnie” względem zbioru uczciwych wniosków, zostaje oznaczony jako podejrzany.

4. Weryfikacja dokumentów (tekst/obraz)
   - GAN-y dobrze radzą sobie z danymi obrazowymi, więc mogą być używane np. do:
     - wykrywania fałszywych skanów dokumentów (dowody osobiste, zaświadczenia lekarskie, akty urodzenia/zgonu),
     - rozpoznawania subtelnych manipulacji (deepfake w dokumentach).

5. Ochrona prywatności (synthetic data)
   - W ubezpieczeniach dane osobowe są bardzo wrażliwe (RODO).
   - GAN-y mogą generować syntetyczne bazy danych klientów,
   - które statystycznie odwzorowują prawdziwe dane, ale nie zawierają danych realnych osób.
   - Takie bazy można bezpiecznie udostępniać do testów i prac rozwojowych.

6. Czynniki potencjalneego ryzyka
   - GAN-y mogą być też użyte przez oszustów do generowania bardzo przekonujących
     fałszywych dokumentów czy deepfake’ów - instytucje muszą być przygotowane na obronę przed tym.

Uwaga: w swoim najważniejszym obszarze zastosowań, mianowicie w grafice,
GAN-y spotkały się w ostatnich czterech latach z silną konkurencją modeli dyfuzyjnych.
Posiadają one jednak istotne zalety, dzięki którym nie zostały wyparte z tych
zastosowań; najważniejsze z nich to szybkość i zdolność do skutecznej pracy
na małych zbiorach danych.
"""

### Poziom trudności: ⭐ (Podstawowy)

Zaczniemy od prostego przykładu pokazującego podstawową strukturę sieci GAN.
Stworzymy generator i dyskryminator dla jednowymiarowych danych.

class SimpleGenerator(nn.Module):
    """
    Prosty generator do tworzenia jednowymiarowych danych.

    W kontekście ubezpieczeniowym może to być np. generator
    wysokości szkód lub wieku klientów.
    """
    def __init__(self, input_size=10, output_size=1):
        super(SimpleGenerator, self).__init__()

        # Sieć składająca się z trzech warstw liniowych
        # input_size: rozmiar wektora szumu (noise)
        # output_size: rozmiar generowanych danych
        self.network = nn.Sequential(
            nn.Linear(input_size, 128),    # Warstwa wejściowa
            nn.ReLU(),                     # Funkcja aktywacji ReLU
            nn.Linear(128, 64),            # Warstwa ukryta
            nn.ReLU(),
            nn.Linear(64, output_size),    # Warstwa wyjściowa
            nn.Tanh()                      # Tanh daje wartości [-1, 1]
        )

    def forward(self, x):
        """Przepuszczenie danych przez sieć"""
        return self.network(x)

class SimpleDiscriminator(nn.Module):
    """
    Prosty dyskryminator oceniający czy dane są prawdziwe czy sztuczne.

    W ubezpieczeniach może oceniać czy profil klienta jest autentyczny.
    """
    def __init__(self, input_size=1):
        super(SimpleDiscriminator, self).__init__()

        # Sieć klasyfikująca dane jako prawdziwe (1) lub fałszywe (0)
        self.network = nn.Sequential(
            nn.Linear(input_size, 64),     # Warstwa wejściowa
            nn.LeakyReLU(0.2),            # LeakyReLU lepsze dla discriminatora
            nn.Linear(64, 32),             # Warstwa ukryta
            nn.LeakyReLU(0.2),
            nn.Linear(32, 1),              # Warstwa wyjściowa (1 neuron)
            nn.Sigmoid()                   # Sigmoid daje prawdopodobieństwo [0,1]
        )

    def forward(self, x):
        """Przepuszczenie danych przez sieć"""
        return self.network(x)

## 3. PRZYKŁAD 2: GAN DO GENEROWANIA DANYCH KLIENTÓW UBEZPIECZENIOWYCH
### Poziom trudności: ⭐⭐ (Łatwy)

Teraz stworzymy bardziej praktyczny przykład - GAN generujący profile klientów ubezpieczeniowych.
"""

def create_insurance_dataset(n_samples=1000):
    """
    Tworzy syntetyczny zbiór danych klientów ubezpieczeniowych.

    Cechy klientów:
    - Wiek (25-80 lat)
    - Dochód roczny (20000-200000)
    - Liczba wypadków w historii (0-5)
    - Staż jazdy w latach (5-50)
    """
    np.random.seed(42)

    # Generowanie realistycznych danych klientów
    age = np.random.normal(45, 15, n_samples)
    age = np.clip(age, 25, 80)  # Ograniczenie wieku do rozsądnych wartości

    # Dochód koreluje z wiekiem (starsi = wyższe zarobki)
    income = 30000 + age * 1000 + np.random.normal(0, 20000, n_samples)
    income = np.clip(income, 20000, 200000)

    # Liczba wypadków - rzadkie zdarzenia
    accidents = np.random.poisson(0.5, n_samples)
    accidents = np.clip(accidents, 0, 5)

    # Staż jazdy związany z wiekiem
    driving_experience = age - 18 + np.random.normal(0, 3, n_samples)
    driving_experience = np.clip(driving_experience, 5, 100)

    # Łączenie danych
    data = np.column_stack([age, income, accidents, driving_experience])

    return data

class InsuranceGenerator(nn.Module):
    """
    Generator profili klientów ubezpieczeniowych.

    Generuje 4-wymiarowe wektory reprezentujące:
    wiek, dochód, liczbę wypadków, staż jazdy
    """
    def __init__(self, noise_dim=100, output_dim=4):
        super(InsuranceGenerator, self).__init__()

        # Większa sieć dla bardziej złożonych danych
        self.network = nn.Sequential(
            nn.Linear(noise_dim, 128),
            nn.BatchNorm1d(128),          # Batch normalization stabilizuje trening
            nn.ReLU(),

            nn.Linear(128, 256),
            nn.BatchNorm1d(256),
            nn.ReLU(),

            nn.Linear(256, 128),
            nn.BatchNorm1d(128),
            nn.ReLU(),

            nn.Linear(128, output_dim),
            nn.Tanh()                     # Dane znormalizowane do [-1, 1]
        )

    def forward(self, noise):
        return self.network(noise)

class InsuranceDiscriminator(nn.Module):
    """
    Dyskryminator oceniający autentyczność profili klientów.
    """
    def __init__(self, input_dim=4):
        super(InsuranceDiscriminator, self).__init__()

        self.network = nn.Sequential(
            nn.Linear(input_dim, 128),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),              # Dropout przeciwdziała overfitting

            nn.Linear(128, 64),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),

            nn.Linear(64, 32),
            nn.LeakyReLU(0.2),

            nn.Linear(32, 1),
            nn.Sigmoid()
        )

    def forward(self, data):
        return self.network(data)

## 4. PRZYKŁAD 3: TRENING GAN - PROCES NAUKI
### Poziom trudności: ⭐⭐⭐ (Średni)

Teraz nauczymy się jak trenować sieć GAN. To najbardziej skomplikowana część prezentacji.
"""

def train_insurance_gan(generator, discriminator, real_data, epochs=100, batch_size=64):
    """
    Funkcja trenująca GAN dla danych ubezpieczeniowych.

    Parametry:
    - generator: sieć generująca dane
    - discriminator: sieć oceniająca dane
    - real_data: prawdziwe dane do nauki
    - epochs: liczba epok treningu
    - batch_size: rozmiar batcha
    """

    # Konwersja do tensorów PyTorch
    real_data_tensor = torch.FloatTensor(real_data)

    # Optymizatory - osobne dla każdej sieci!
    # Algorytm Adam działa dobrze dla GAN
    optimizer_G = optim.Adam(generator.parameters(), lr=0.0002, betas=(0.5, 0.999))
    optimizer_D = optim.Adam(discriminator.parameters(), lr=0.0002, betas=(0.5, 0.999))

    # Funkcja straty - Binary Cross Entropy
    criterion = nn.BCELoss()

    # Listy do przechowywania historii treningu
    d_losses = []
    g_losses = []

    print("🚀 Rozpoczynanie treningu GAN...")
    print("📊 Format: [Epoka] D_loss: X.XXX, G_loss: X.XXX")

    for epoch in range(epochs):
        # Liczniki strat w epoce
        d_loss_epoch = 0
        g_loss_epoch = 0
        num_batches = 0

        # Trening w batchach
        for i in range(0, len(real_data_tensor), batch_size):
            # Pobieranie batcha prawdziwych danych
            real_batch = real_data_tensor[i:i+batch_size]
            batch_size_actual = real_batch.size(0)

            # === TRENING DYSKRYMINATORA ===
            optimizer_D.zero_grad()

            # Etykiety: 1 = prawdziwe, 0 = fałszywe
            real_labels = torch.ones(batch_size_actual, 1)
            fake_labels = torch.zeros(batch_size_actual, 1)

            # Ocena prawdziwych danych
            real_predictions = discriminator(real_batch)
            real_loss = criterion(real_predictions, real_labels)

            # Generowanie fałszywych danych
            noise = torch.randn(batch_size_actual, 100)
            fake_data = generator(noise)

            # Ocena fałszywych danych (detach() żeby nie trenować generatora!)
            fake_predictions = discriminator(fake_data.detach())
            fake_loss = criterion(fake_predictions, fake_labels)

            # Łączna strata discriminatora
            d_loss = real_loss + fake_loss
            d_loss.backward()
            optimizer_D.step()

            # === TRENING GENERATORA ===
            optimizer_G.zero_grad()

            # Generator chce, żeby dyskryminator uznał jego dane za prawdziwe
            # Dlatego używamy real_labels (1) jako target!
            fake_predictions = discriminator(fake_data)
            g_loss = criterion(fake_predictions, real_labels)

            g_loss.backward()
            optimizer_G.step()

            # Akumulacja strat
            d_loss_epoch += d_loss.item()
            g_loss_epoch += g_loss.item()
            num_batches += 1

        # Średnie straty w epoce
        d_loss_avg = d_loss_epoch / num_batches
        g_loss_avg = g_loss_epoch / num_batches

        d_losses.append(d_loss_avg)
        g_losses.append(g_loss_avg)

        # Wyświetlanie postępu co 10 epok
        if epoch % 10 == 0:
            print(f"[{epoch:3d}] D_loss: {d_loss_avg:.3f}, G_loss: {g_loss_avg:.3f}")

    print("✅ Trening zakończony!")
    return d_losses, g_losses

# URUCHOMIENIE TRENINGU

print("⚠️  UWAGA: Trening może potrwać 2-5 minut...")

"""
### Istotne obserwacje wynikające z treningu:

1. **Balans strat**: Generator i dyskryminator powinny "walczyć" równomiernie
2. **Stabilność**: Duże wahania mogą oznaczać problemy z treningiem
3. **Jakość danych**: Wygenerowane dane powinny przypominać prawdziwe

### Praktyczne zastosowania:

- **Stress testing**: Generowanie ekstremalnych profili klientów
- **Regulacje GDPR**: Zastępowanie prawdziwych danych syntetycznymi
- **A/B testing**: Tworzenie kontrolowanych grup testowych

---

## 5. PRZYKŁAD 4: GAN DO WYKRYWANIA OSZUSTW UBEZPIECZENIOWYCH
### Poziom trudności: ⭐⭐⭐⭐ (Trudny)

Przykład - używamy GAN do generowania przykładów oszustw ubezpieczeniowych!
"""

def create_fraud_dataset(n_samples=1000, fraud_ratio=0.1):
    """
    Tworzy zbiór danych z prawdziwymi i fałszywymi roszczeniami ubezpieczeniowymi.

    Cechy roszczenia:
    - Wartość szkody
    - Wiek pojazdu
    - Doświadczenie kierowcy
    - Lokalizacja (kod pocztowy)
    - Czas od polisy do szkody
    - Dzień tygodnia zdarzenia
    """
    np.random.seed(42)

    n_fraud = int(n_samples * fraud_ratio)
    n_legitimate = n_samples - n_fraud

    # === PRAWDZIWE ROSZCZENIA ===
    # Normalne szkody - rozkład logarytmiczno-normalny
    legitimate_damage = np.random.lognormal(8, 1, n_legitimate)  # ~3k średnia
    legitimate_damage = np.clip(legitimate_damage, 500, 50000)

    # Wiek pojazdu - normalny rozkład
    legitimate_age = np.random.normal(7, 4, n_legitimate)
    legitimate_age = np.clip(legitimate_age, 1, 20)

    # Doświadczenie kierowcy
    legitimate_exp = np.random.normal(15, 8, n_legitimate)
    legitimate_exp = np.clip(legitimate_exp, 1, 40)

    # Kod pocztowy (symulowany)
    legitimate_postal = np.random.randint(10000, 99999, n_legitimate)

    # Czas od polisy - większość w pierwszych latach
    legitimate_time = np.random.exponential(1.5, n_legitimate)
    legitimate_time = np.clip(legitimate_time, 0.1, 5)

    # Dzień tygodnia (1-7)
    legitimate_weekday = np.random.randint(1, 8, n_legitimate)

    # === OSZUKAŃCZE ROSZCZENIA ===
    # Wyższe szkody, podejrzane wzorce
    fraud_damage = np.random.lognormal(9.5, 0.8, n_fraud)  # ~13k średnia!
    fraud_damage = np.clip(fraud_damage, 5000, 100000)

    # Starsze pojazdy (łatwiej oszukać)
    fraud_age = np.random.normal(12, 3, n_fraud)

class FraudGAN_Generator(nn.Module):
    """
    Generator do tworzenia syntetycznych przykładów oszustw.

    Pomaga w treningu systemów wykrywających oszustwa poprzez

        super(FraudGAN_Generator, self).__init__()

        self.network = nn.Sequential(
            nn.Linear(noise_dim, 256),
            nn.BatchNorm1d(256),
            nn.LeakyReLU(0.2),

            nn.BatchNorm1d(512),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.2),

            nn.Linear(512, 256),
            nn.BatchNorm1d(256),
            nn.LeakyReLU(0.2),

            nn.Linear(256, output_dim),
            nn.Tanh()
        )

class FraudGAN_Discriminator(nn.Module):

        super(FraudGAN_Discriminator, self).__init__()

            nn.Dropout(0.4),

            nn.Linear(256, 128),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),

            nn.Linear(128, 1),

        )

def calculate_business_value(real_data, synthetic_data, fraud_detection_improvement):
    """
    Oblicza wartość biznesową implementacji GAN w firmie ubezpieczeniowej.

    Parametry:
    - real_data: prawdziwe dane
    - synthetic_data: dane wygenerowane przez GAN
    - fraud_detection_improvement: poprawa wykrywania oszustw (%)
    """

    # Symulowane wartości biznesowe firmy ubezpieczeniowej
    annual_claims_volume = 50000  # roczna liczba roszczeń
    average_claim_value = 8000    # średnia wartość roszczenia (PLN)
    fraud_rate = 0.12            # 12% roszczeń to oszustwa
    current_detection_rate = 0.65 # obecne wykrywanie 65% oszustw

    # Obliczenia bazowe
    annual_claims_value = annual_claims_volume * average_claim_value
    annual_fraud_value = annual_claims_value * fraud_rate
    currently_detected_fraud = annual_fraud_value * current_detection_rate
    currently_undetected_fraud = annual_fraud_value - currently_detected_fraud

    # Poprawa dzięki GAN
    new_detection_rate = min(0.95, current_detection_rate + fraud_detection_improvement)
    newly_detected_fraud = annual_fraud_value * new_detection_rate
    additional_savings = newly_detected_fraud - currently_detected_fraud

    # Koszty implementacji
    implementation_cost = 500000   # PLN - jednorazowy koszt
    annual_maintenance = 100000    # PLN - roczne utrzymanie

    # ROI calculation
    net_annual_benefit = additional_savings - annual_maintenance
    roi_year_1 = (net_annual_benefit - implementation_cost) / implementation_cost * 100
    roi_year_3 = ((net_annual_benefit * 3) - implementation_cost) / implementation_cost * 100

    results = {
        'Roczna wartość roszczeń': annual_claims_value,
        'Roczna wartość oszustw': annual_fraud_value,
        'Obecnie wykrywane oszustwa': currently_detected_fraud,
        'Nowa stopa wykrywania': f"{new_detection_rate:.1%}",
        'Dodatkowe oszczędności roczne': additional_savings,
        'Koszt implementacji': implementation_cost,
        'Roczne utrzymanie': annual_maintenance,
        'ROI rok 1': f"{roi_year_1:.1f}%",
        'ROI rok 3': f"{roi_year_3:.1f}%",
        'Payback period': implementation_cost / net_annual_benefit
    }

    return results

def evaluate_gan_quality(real_data, synthetic_data, feature_names):
    """
    Ocenia jakość danych wygenerowanych przez GAN używając różnych metryk.
    """
    from scipy import stats

    print("🔍 OCENA JAKOŚCI DANYCH WYGENEROWANYCH PRZEZ GAN")
    print("=" * 60)

    quality_scores = {}

    # 1. Porównanie rozkładów statystycznych
    print("\n📊 1. ANALIZA ROZKŁADÓW STATYSTYCZNYCH")
    for i, feature in enumerate(feature_names):
        real_feature = real_data[:, i]
        synthetic_feature = synthetic_data[:, i]

        # Test Kołmogorowa-Smirnowa
        ks_statistic, ks_p_value = stats.ks_2samp(real_feature, synthetic_feature)

        # Porównanie momentów
        real_mean, real_std = np.mean(real_feature), np.std(real_feature)
        synth_mean, synth_std = np.mean(synthetic_feature), np.std(synthetic_feature)

        mean_diff = abs(real_mean - synth_mean) / real_mean * 100
        std_diff = abs(real_std - synth_std) / real_std * 100

        print(f"\n{feature}:")
        print(f"  KS test p-value: {ks_p_value:.4f} {'✅' if ks_p_value > 0.05 else '❌'}")
        print(f"  Różnica średnich: {mean_diff:.1f}%")
        print(f"  Różnica odchyleń: {std_diff:.1f}%")

        # Scoring (0-100)
        ks_score = 100 if ks_p_value > 0.05 else max(0, 100 * ks_p_value / 0.05)
        mean_score = max(0, 100 - mean_diff)
        std_score = max(0, 100 - std_diff)

        feature_score = (ks_score + mean_score + std_score) / 3
        quality_scores[feature] = feature_score

    # 2. Analiza korelacji
    print(f"\n📈 2. ANALIZA KORELACJI")
    real_corr = np.corrcoef(real_data.T)
    synthetic_corr = np.corrcoef(synthetic_data.T)

    corr_diff = np.abs(real_corr - synthetic_corr)
    avg_corr_diff = np.mean(corr_diff[np.triu_indices_from(corr_diff, k=1)])

    print(f"Średnia różnica korelacji: {avg_corr_diff:.3f}")
    print(f"Ocena korelacji: {'✅' if avg_corr_diff < 0.1 else '❌'}")

    # 3. Wykrywanie outlierów
    print(f"\n🎯 3. ANALIZA OUTLIERÓW")

    def count_outliers(data):
        Q1 = np.percentile(data, 25, axis=0)
        Q3 = np.percentile(data, 75, axis=0)
        IQR = Q3 - Q1
        lower_bound = Q1 - 1.5 * IQR
        upper_bound = Q3 + 1.5 * IQR
        outliers = np.sum((data < lower_bound) | (data > upper_bound))
        return outliers

    real_outliers = count_outliers(real_data)
    synthetic_outliers = count_outliers(synthetic_data)

    outlier_ratio = synthetic_outliers / real_outliers if real_outliers > 0 else 1
    print(f"Outliers w prawdziwych danych: {real_outliers}")
    print(f"Outliers w syntetycznych danych: {synthetic_outliers}")
    print(f"Stosunek outlierów: {outlier_ratio:.2f}")

    # 4. Ocena ogólna
    overall_quality = np.mean(list(quality_scores.values()))

    print(f"\n OCENA OGÓLNA")
    print(f"Jakość danych GAN: {overall_quality:.1f}/100")

    if overall_quality >= 80:
        grade = " Doskonała - gotowa do produkcji"
    elif overall_quality >= 70:
        grade = " Bardzo dobra - wymaga drobnych poprawek"
    elif overall_quality >= 60:
        grade = " Dobra - wymaga optymalizacji"
    else:
        grade = " Niezadowalająca - wymaga przebudowy"

    print(f"Rekomendacja: {grade}")

    return quality_scores, overall_quality
